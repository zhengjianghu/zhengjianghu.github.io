单片机基础之“ROM&RAM” 对单片机中的ROM和RAM做一个简单的研究与知识汇总，后续可能还会有补充！

ROM&RAM详解

前言

ROM（程序存储器）和RAM（数据存储器）作为单片机中内部结构中必不可少的模块，担任着所有的存储功能，同时其容量的大小，也是单片机性能的体现之一。 如果想深究单片机的内部工作原理，那么ROM和RAM是一道绕不开的坎。以下（MCS-51单片机）就是个人的一点理解与学习笔记汇总。

ROM

概念

ROM(Read only Memory)：只读存储器，在出厂以后只能读取而不能写入，一般用来作为程序存储器存储固件或者程序。
特征：掉电不丢失数据，每次上电之后程序又从头开始运行，其中的程序在产品出厂前就写好，用户无法修改。

大小及结构

ROM可分为片内ROM和片外ROM。片内ROM只满足基本程序的大小需求，当用户的程序需要更大的存储空间时， 可以选择外接程序存储器来扩展程序。

片内ROM：4KB字节（000H-FFFH）、寻址指令：MOVC
片外ROM：64KB字节（0000H-FFFFH）、寻址指令：MOVC

关于ROM的可寻址范围：由于ROM的程序地址寄存器是16位寻址，因此最大可以实现65536(1111111111111111)字节，即大约64K字节的存储器寻址。但是 单片机内部由于成本等原因只配置4K字节的空间，但是外部配置便可以实现最大化的寻址。存储器的最大容量，取决于可寻址长度，当可寻址长度成为4字节32位时 就可以配置4294967296（11111111111111111111111111111111）字节的ROM存储器。

ROM的内部结构分布：
0000H：程序的第一条指令，复位时程序计数器所指向的地址
0003F：INT0中断入口
000BH：T0中断入口
0013H：INT1中断入口
001BH：T1中断入口
0023H：串行中断入口
0100H：主程序

由存储分布可知，开始执行程序时，并不是按照地址的顺序依次寻址取指令，需要先跳过前面的几个中断地址入口，直接到达0100H的 主程序入口，再开始执行主程序，而各中断服务程序都存放在各自的地址范围内，比如：
INT0中断程序：0003H--000AH; 7字节
T0中断程序：000BH--0012H； 7字节
ITN1中断程序：0013H--001AH; 7字节
T1中断程序：01BH--0022H; 7字节
串行中断程序：023H--00FFH； 237字节

RAM

概念

RAM（Read Access Memory）：随机存储器，当程序被写入之后可以进行更改与擦除。断电之后数据会丢失 当下次上电时，需要重新运行程序来生成数据。
举例：手机内部的内存既可以下载软件占用内存，也可以通过卸载软件的方式来重新获得内存，该内存便属于EAM存储。

大小及内部结构

RAM和ROM一样，可分为片内RAM和片外RAM。
由于在单片机内部，数据地址寄存器为8位寻址，因此最大只能进行256B字节的RAM寻址，但是当进行片外寻址时， 由于是16位寻址，因此可以进行最大64K字节的RAM寻址。
片内：256B字节； 00H-FFH; 寻址指令：MOV
片外：64K字节；0000H-FFFF； 寻址指令： MOVX
RAM的内部存储分布（片内RAM为例）：

00H--7FH：(0--128)低128位，通用数据存储器
{
1、{
00H-07H：R0(第一组工作寄存器，4*8=32 B)
08H-0FH：R1(第二组工作寄存器，4*8=32 B)
10H-17H：R2(第三组工作寄存器，4*8=32 B)
18H-20H：R3(第三组工作寄存器，4*8=32 B)
}
2、 {
20H-2FH：位寻址区（16 B）
}
3、{
30H-7FH：便笺区，通用数据区（80 B）
}
}
32+16+80=128 B

80FH--FFH：（129-256）高128位，属于SFR（Special Function Register）特殊功能寄存器
{
只用了21个寄存器，剩余的107个没用
}
21+107=128 B
128+128=256 B
