Linux开机自动运行程序

Linux开机启动过程

  1. BIOS
    2. 硬件检测，检查CPU和内存、由于在BIOS中存储了各磁盘的启动顺序，因此BIOS会按照启动顺序依次在各个盘中查找 MBR信息，找到之后并将其加载到内存中，之后控制权转交给Boot Loader程序.
  2. MBR
    3. MBR(Master Boot Record)：主引导记录，存储Boot Loader程序，开始加载GRUB
  3. GRUB
    4. GRUB(Grand Unified Bootloader)：多系统启动程序，主要工作有三步：step1加载查找第二段Bootloader程序（就是step2）、step2识别文件系统 、step3查找并加载Kernel的信息，然后把控制权交给Kernel。
  4. Kernel
    5. Kernel：内核。将Kernel读进内存并将压缩形式的Kernel进行解压，从之前就被加载进内存中的根文件系统initrd文件中加载驱动模块，完毕之后将initrd删除。 挂载完根文件系统后，开始装载第一个进程，执行/sbin/init，之后将控制权交给init程序。
  5. Init
    6. Init：OS的初始化操作。该初始化并不是直接执行一个叫做init的文件，而是根据/etc/inittab（定义了系统默认运行级别）设定的动作进行脚本的执行，而在 这其中，第一个被执行的脚本为/etc/rc.d/rc.sysinit，这个才是真正的OS初始化脚本。
    执行完毕后，根据配置的启动级别，执行对应目录下的脚本，最后执行/etc/rc.d/rc.local这个脚本，至此，系统启动完成。
